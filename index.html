网络隐身和洋葱路由

发表于 2015-04-18   |   分类于 渗透测试   |     |   阅读次数

在渗透测试中的必要性

  • 要访问的资源可能是非法的恶意资源
  • 尽量隐藏渗透测试过程中的测试或攻击来源

对于一个渗透测试人员来说，一个虚拟的IP地址对于模拟一个真实的攻击者来说是非常重要的。例如在侦察阶段，主动侦察会暴露自己的行迹，一个熟练的攻击者不会使用相同的IP来扫描一个网站；端口扫描阶段，不断地扫描目标网站的服务类型必定也会引起管理员的注意。攻击者会使用一些假IP来扫描目标，然后使用其他IP来执行渗透。

实现网络隐身的方法

设置代理跳板

什么是“代理”

代理（Proxy），也称网络代理，是一种特殊的网络服务，允许一个网络终端（一般为客户端）通过这个服务与另一个网络终端（一般为服务器）进行非直接的连接。提供代理服务的电脑系统或其它类型的网络终端称为代理服务器。接入该服务器后可让计算机“脱离”自己所在的网络范围，本机的上网流量全部通过代理服务器发送，因此 IP 地址得以遮蔽，所有上网流量看上去似乎都来自代理服务器。

渗透测试（2）——侦察

发表于 2015-04-18   |   分类于 渗透测试   |     |   阅读次数

侦察，以彻底地搜索目标的公共信息作为开始。

主要任务

第一，收集与目标有关的信息，越多越好；第二，分类所以收集的信息，创建可进行攻击的IP地址列表。

策略：

主动侦察

包括与目标系统的直接交互。NOTICE：这个过程中目标可能会记录下我们的IP地址及活动。

一般情况下，侦察阶段首先要做的是仔细浏览目标网站。可能会用到的工具：HTTrack 网站复制机——能够创建与目标网站完全相同的脱机副本，复制的内容包括原始网站所有的网页、链接、图片和代码。这些文件将储存在你的本地电脑里。

tips：关注细节，例如目标的物理地址、位置、电话号码、email地址、运营时间、员工资料、新闻公告、招聘启事等。

被动侦察

利用从网上获取的海量信息。当执行被动侦察的时候，我们不会直接目标交互，因此目标也不可能知道或记录我们的活动。首先要做的是利用各个搜索引擎对目标进行地毯式搜索。【渗透测试人员应该知道的google黑客技术】

可能用到的工具和方法：

  • google搜索引擎
  • The Harvester：挖掘并利用邮箱地址
  • Whois
  • Netcraft
  • host工具
  • 从DNS中提取信息：NS Lookup、 Dig
  • 从电子邮件服务器中提取信息
  • MetaGooFil：用来提取元数据
  • 社会工程学
  • 筛选信息以寻找可攻击的目标

渗透测试方法论

发表于 2015-04-18   |   分类于 渗透测试   |     |   阅读次数

简介

渗透测试是一种合法且授权定位计算机系统，并对其成功实施漏洞攻击的方法，其目的是为了使这些受测系统更加安全。

渗透测试（Penetration Testing或者Pen Testing，PT）也称为：

  • 黑客活动（Hacking）
  • 道德黑客（Ethical Hacking）
  • 白帽黑客（White Hat Hacking）

搭建平台

Linux中用于渗透测试的最常用的发行版之一是“BackTrack”（现在，BackTrack已被Kali Linux所代替，BT将不再维护），它预先加载了成百上千种安装并配置好的安全工具。默认情况下，Backtrack并不启用网络功能，因为作为渗透测试人员通常会设法隐藏自己来避免被人发现，计算机一旦接入网络，启动之后就会马上通过广播的方式向整个网络申请DHCP服务器和IP地址，暴露行踪。为了避免这种情况，Backtrack机器的网络接口默认是关闭的。

启用网络（kali linux）：【ifconfig用于临时设置，重启失效】
ifconfig -a
ifconfig eth0 up

手动配置IP地址：
ifconfig eth0 [IP] netmask [子网掩码]

设置网关：
route add default gw 网关地址
route add default dev eth0 //默认路由，将去往未知网络的数据包全部从接口eth0发出去

使用DHCP分配IP地址：
dhclient eth0

黑客渗透测试入门方法论

侦察

也就是收集信息，在收集目标信息上所花的时间越多，后续阶段的成功率就越高。而这一步骤恰恰是当前整个渗透测试方法体系中最容易被忽略、最不被重视、最易受人误解的一环。

扫描

  • 端口扫描
  • 漏洞扫描

一旦完成了端口扫描，就拥有了目标的开放端口和可能已开启服务的列表；漏洞扫描用于定位和识别在目标计算机上运行的软件和服务所存在的缺陷。

漏洞利用

漏洞利用的最终目标是在目标计算机上获得管理员权限（即完全控制权）。

维持访问

通常，在漏洞利用阶段提供的攻击载荷只能提供开启系统的临时访问权限，所以需要创建一个可以永久控制目标系统的后门进程。这个后门进程可以让我们以管理员的权限重新开启已经关闭的程序，甚至重启目标计算机。

TinyOS 节点‐节点无线通信

发表于 2015-02-15   |   分类于 TinyOS   |     |   阅读次数
本节介绍如何在 TinyOS 上进行节点与节点之间的无线通信。打开例子程序 BlinkToRadio,这个程序通过消息发送自身的计数器至对方,同时收到对方的消息后,解析出对方的计数器,按照这个计数器亮灯,使用单个 Timer 实现发送的频率间隔。

主动消息概述

在TinyOS系统中，无线通信模型是基于主动消息模式的通信模型。在无线传感器网络中采用主动消息机制主要目的是使无线传感器节点的计算和通信重叠。

相关组件及接口

TinyOS提供了很多与底层通信相关的接口，并提供了实现这些接口的组件。其中最常用的有AMSenderC，AMReceiverC，ActiveMessageC等组件。这些接口组件都使用一个共同的消息缓存区，称为message_t。

message_t

这是一个nesC的结构体，其定义可在/opt/tinyos-2.1.1/tos/types目录下的message.h文件中看到，具体如下：

其中有4个元素：

  • header: 消息的头部。
  • data: 有效荷载区，即数据。
  • footer: 消息的尾部。
  • metadata: 元数据。

注意：header , footer , metadata 都是不透明的，不能直接访问。要访问 message_t 必须通过Packet , AMPacket和其他的一些接口。

  • AMPacket接口：类似Packet，提供对message_t抽象数据类型的基本的AM 访问。这个接口提供的命令有：获得节点的AM 地址，AM信息包的目标地址以及 AM信息包的类型。除此之外还有，设置 AM信息包目标地址和类型，检查目标地址是否为本地节点。
