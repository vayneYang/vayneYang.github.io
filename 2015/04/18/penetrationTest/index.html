渗透测试方法论

blueblue
2015-04-18
渗透测试
方法论

简介

渗透测试是一种合法且授权定位计算机系统，并对其成功实施漏洞攻击的方法，其目的是为了使这些受测系统更加安全。

渗透测试（Penetration Testing或者Pen Testing，PT）也称为：

  • 黑客活动（Hacking）
  • 道德黑客（Ethical Hacking）
  • 白帽黑客（White Hat Hacking）

搭建平台

Linux中用于渗透测试的最常用的发行版之一是“BackTrack”（现在，BackTrack已被Kali Linux所代替，BT将不再维护），它预先加载了成百上千种安装并配置好的安全工具。默认情况下，Backtrack并不启用网络功能，因为作为渗透测试人员通常会设法隐藏自己来避免被人发现，计算机一旦接入网络，启动之后就会马上通过广播的方式向整个网络申请DHCP服务器和IP地址，暴露行踪。为了避免这种情况，Backtrack机器的网络接口默认是关闭的。

启用网络（kali linux）：【ifconfig用于临时设置，重启失效】
ifconfig -a
ifconfig eth0 up

手动配置IP地址：
ifconfig eth0 [IP] netmask [子网掩码]

设置网关：
route add default gw 网关地址
route add default dev eth0 //默认路由，将去往未知网络的数据包全部从接口eth0发出去

使用DHCP分配IP地址：
dhclient eth0

黑客渗透测试入门方法论

侦察

也就是收集信息，在收集目标信息上所花的时间越多，后续阶段的成功率就越高。而这一步骤恰恰是当前整个渗透测试方法体系中最容易被忽略、最不被重视、最易受人误解的一环。

扫描

  • 端口扫描
  • 漏洞扫描

一旦完成了端口扫描，就拥有了目标的开放端口和可能已开启服务的列表；漏洞扫描用于定位和识别在目标计算机上运行的软件和服务所存在的缺陷。

漏洞利用

漏洞利用的最终目标是在目标计算机上获得管理员权限（即完全控制权）。

维持访问

通常，在漏洞利用阶段提供的攻击载荷只能提供开启系统的临时访问权限，所以需要创建一个可以永久控制目标系统的后门进程。这个后门进程可以让我们以管理员的权限重新开启已经关闭的程序，甚至重启目标计算机。
